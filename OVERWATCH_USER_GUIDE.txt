<h1 id="what-is">1. What Is Overwatch?</h1>

<p>Overwatch is the internal monitoring control plane for your infrastructure. It sits in front of a Zabbix monitoring server and adds:</p>
<ul>
  <li><strong>Role-based access control</strong> — operators, team leads, admins, and auditors each see different things</li>
  <li><strong>Team management</strong> — group users into teams with assigned hosts</li>
  <li><strong>Simplified workflows</strong> — add monitoring to a host, define custom alert rules, bulk-import hosts from CSV/Excel, and export inventory</li>
  <li><strong>Live problem feed</strong> — real-time alerts with sound notifications and acknowledgement</li>
</ul>

<div class="info">
  <div class="panel-title">Important</div>
  All data comes from Zabbix in real time. Overwatch does not store metrics itself — it is a control and visibility layer on top of Zabbix.
</div>

<hr>

<h1 id="login">2. Logging In</h1>

<ol>
  <li>Open the Overwatch URL in your browser.</li>
  <li>Enter your <strong>username</strong> and <strong>password</strong>.</li>
  <li>Click <strong>Sign In</strong>.</li>
</ol>

<div class="note">
  <div class="panel-title">Forgotten password?</div>
  Passwords can only be reset by an administrator. Contact your team lead or an admin — they can set a new password from the Users page.
</div>

<hr>

<h1 id="interface">3. The Interface</h1>

<h2>Sidebar</h2>
<p>The left sidebar is your main navigation.</p>

<table>
  <tr><th>Element</th><th>Description</th></tr>
  <tr><td><strong>Overwatch logo</strong></td><td>Always visible at the top</td></tr>
  <tr><td><strong>Dark / Light mode toggle</strong></td><td>Moon/sun icon next to the logo</td></tr>
  <tr><td><strong>Navigation groups</strong></td><td>Expandable sections — click a group header to expand or collapse</td></tr>
  <tr><td><strong>Notification Center</strong></td><td>Inbox icon — opens your full alert history</td></tr>
  <tr><td><strong>Active problems count</strong></td><td>Shows unacknowledged Zabbix problems; turns red when problems exist</td></tr>
  <tr><td><strong>Sound controls</strong></td><td>Mute/unmute button and music-note icon for choosing the alert sound</td></tr>
  <tr><td><strong>Backend API / Zabbix dots</strong></td><td>Green = connected, Red = unreachable — at the bottom of the sidebar</td></tr>
  <tr><td><strong>Your username and role</strong></td><td>At the very bottom, with a logout button</td></tr>
</table>

<h2>Status Dots</h2>
<p>The two coloured dots at the bottom of the sidebar update every <strong>10 seconds</strong>:</p>
<ul>
  <li><strong>Backend API</strong> — green means the Overwatch backend is running and reachable</li>
  <li><strong>Zabbix</strong> — green means the backend can reach your Zabbix server</li>
</ul>

<div class="warning">
  <div class="panel-title">Red status dot?</div>
  If either dot is red, data across the portal will be stale or unavailable. See the Troubleshooting section at the bottom of this page.
</div>

<h2>Data Refresh</h2>
<p>All pages refresh automatically every <strong>10 seconds</strong>. You do not need to manually refresh your browser.</p>

<h1 id="roles">4. Roles and Permissions</h1>

<table>
  <tr><th>Role</th><th>What they can do</th></tr>
  <tr><td><strong>Root</strong></td><td>Full access to everything. Can manage all users, teams, and system settings. Can grant any role.</td></tr>
  <tr><td><strong>Team Lead</strong></td><td>Can manage users within their team, assign hosts, and see all admin sections. Cannot manage other team leads or root users.</td></tr>
  <tr><td><strong>Operator</strong></td><td>Can view hosts, metrics, and problems and acknowledge alerts. Cannot create or delete users or change system settings.</td></tr>
  <tr><td><strong>Auditor</strong></td><td>Read-only access. Can view everything but cannot make changes or acknowledge problems. Only Root can grant this role.</td></tr>
</table>

<div class="info">
  The admin sections (Data Collection, Alerts, Users, Administration) are only visible to <strong>Root</strong> and <strong>Team Lead</strong> roles.
</div>

<hr>

<h1 id="sections">5. Section Guide</h1>

<h2 id="overview">Overview</h2>
<p>The overview page is your main health dashboard. It shows:</p>
<ul>
  <li><strong>Total hosts</strong> and how many are currently available vs. unreachable</li>
  <li><strong>Active problems</strong> count broken down by severity</li>
  <li><strong>Recent problems</strong> — a live list of the most recent Zabbix trigger events</li>
  <li><strong>Per-team summary</strong> — how many hosts and active problems each team owns</li>
</ul>

<hr>

<h2 id="hosts">Hosts</h2>

<h3>Adding a host</h3>
<ol>
  <li>Click <strong>Add Host</strong> (top right).</li>
  <li>Fill in: Hostname, IP address, Host group (required), and Templates (optional).</li>
  <li>Click <strong>Save</strong>.</li>
</ol>

<h3>Importing hosts from CSV or Excel</h3>
<ol>
  <li>Click <strong>Import</strong>.</li>
  <li>Download the template first to see the required column format.</li>
  <li>Required columns: <code>host</code> and <code>ip</code>. Optional: <code>group</code>, <code>templates</code>, <code>description</code>.</li>
  <li>Upload your file and confirm.</li>
</ol>

<h3>Exporting hosts</h3>
<p>Click <strong>Export</strong>, then choose <strong>Excel (.xlsx)</strong> or <strong>CSV (.csv)</strong>. The export includes: Hostname, IP, DNS, Status, Availability, Proxy, Host Groups, Templates, and Description.</p>

<h3>Deleting a host</h3>
<p>Click the delete icon on the host row. A confirmation dialog will appear.</p>
<div class="warning">
  <div class="panel-title">Deletion is permanent</div>
  Deleting a host removes it from Zabbix. All associated items, triggers, and history are deleted and cannot be recovered.
</div>

<hr>

<h2 id="dashboard">Dashboard</h2>
<p>Select a host from the dropdown to view its live Zabbix graphs (CPU, memory, network traffic, and any other metrics with graph definitions). Graphs update in real time.</p>
<div class="note">If no graphs appear, the host either has no graph definitions configured in Zabbix or is currently unreachable.</div>

<hr>

<h2 id="monitoring">Monitoring</h2>

<h3>Problems</h3>
<p>Shows all currently active Zabbix trigger problems. Severity is colour-coded:</p>
<table>
  <tr><th>Colour</th><th>Severity</th></tr>
  <tr><td>Dark red</td><td>Critical</td></tr>
  <tr><td>Red</td><td>High</td></tr>
  <tr><td>Orange</td><td>Medium</td></tr>
  <tr><td>Yellow</td><td>Low</td></tr>
  <tr><td>Blue</td><td>Info</td></tr>
</table>

<p><strong>Click any row</strong> to expand it and see: full host detail, duration, who acknowledged it, when, and the acknowledgement note.</p>

<h3>Acknowledging a problem</h3>
<ol>
  <li>Find the problem in the table.</li>
  <li>Click the <strong>Ack</strong> button on the right side of the row.</li>
  <li>Enter an optional note.</li>
  <li>Click <strong>Confirm</strong>.</li>
</ol>
<div class="info">Acknowledging marks the problem in Zabbix and removes it from the sidebar badge. The problem stays open until its trigger condition clears in Zabbix.</div>

<h3>Latest Data</h3>
<p>Shows the last collected value for every monitored item on a selected host. Choose a host from the dropdown, then use the search box to filter.</p>
<table>
  <tr><th>Status chip</th><th>Meaning</th></tr>
  <tr><td><strong>Enabled</strong> (green)</td><td>Item is active and collecting</td></tr>
  <tr><td><strong>Disabled</strong> (grey)</td><td>Item exists but is paused</td></tr>
  <tr><td><strong>No data</strong> (red)</td><td>Item has not reported within its expected interval — usually means the host agent is unreachable</td></tr>
</table>
<div class="warning">A yellow banner appears at the top of the table when the host agent is unreachable. All items will show <strong>No data</strong> until the agent comes back online.</div>

<h3>Items</h3>
<p>Manages monitoring items on hosts. Select a host to see all its items. <strong>Click any row</strong> to expand and see: type, interval, source, full key, and exact last-collected timestamp.</p>

<h3>Adding an item</h3>
<ol>
  <li>Select a host.</li>
  <li>Click <strong>Add Item</strong>.</li>
  <li>Fill in: name, key (e.g. <code>agent.ping</code>), type, value type, and polling interval.</li>
  <li>Click <strong>Save</strong>.</li>
</ol>

<h3>Triggers</h3>
<p>A trigger is a condition on an item that creates a problem when true (e.g. "alert if CPU &gt; 90% for 5 minutes").</p>
<ol>
  <li>Select a host and item.</li>
  <li>Set the condition operator and threshold.</li>
  <li>Set the severity.</li>
  <li>Click <strong>Create Trigger</strong>.</li>
</ol>

<h3>Graphs</h3>
<p>Time-series charts for item values. Select a host and an item, then choose a time range to view historical data.</p>

<h2 id="services">Services &amp; SLA</h2>

<h3>Services</h3>
<p>Lists all Zabbix services (logical business nodes such as "Payment API" or "Database Cluster"). You can create, edit, and delete services here.</p>

<h3>SLA</h3>
<p>Lists SLA definitions. Each SLA has a target uptime percentage, a measurement period, and service tags that define which services it applies to.</p>
<div class="note">SLA is a Zabbix 6.0+ feature. If your Zabbix version is older, the SLA tab will always be empty.</div>

<h2 id="alerts">Alerts <em>(Admin only)</em></h2>

<h3>Alert Rules</h3>
<p>Custom threshold-based alerts defined in Overwatch (separate from Zabbix triggers). These run on a 60-second check loop.</p>
<ol>
  <li>Click <strong>Add Rule</strong>.</li>
  <li>Select the host and item to watch.</li>
  <li>Choose the operator (<code>&gt;</code>, <code>&lt;</code>, <code>=</code>, etc.) and threshold value.</li>
  <li>Set the severity.</li>
  <li>Optionally assign a custom alert sound for this rule.</li>
  <li>Click <strong>Save</strong>.</li>
</ol>
<div class="tip">When the rule fires, a notification popup appears in the bottom-right corner and is logged in the Notification Center.</div>

<h3>Other alert sub-pages</h3>
<table>
  <tr><th>Sub-page</th><th>What it does</th></tr>
  <tr><td><strong>Trigger / Service / Discovery Actions</strong></td><td>What Zabbix does automatically when a trigger fires (send email, run script, etc.)</td></tr>
  <tr><td><strong>Autoregistration</strong></td><td>Rules for auto-adding newly discovered hosts</td></tr>
  <tr><td><strong>Media Types</strong></td><td>Email servers, webhook URLs, and SMS gateways configured in Zabbix</td></tr>
  <tr><td><strong>Scripts</strong></td><td>Zabbix global scripts that can be run against hosts</td></tr>
</table>

<h1 id="notifications">6. Notification System</h1>

<h2>Popup Notifications</h2>
<p>When a new problem fires, a notification card appears in the <strong>bottom-right corner</strong>. Each card shows: severity, how long ago it fired, host name, problem description, and an <strong>Acknowledge</strong> button.</p>
<ul>
  <li>Low/Info severity notifications auto-dismiss after <strong>8 seconds</strong></li>
  <li>Critical/High/Medium stay until you dismiss them</li>
  <li>If more than 3 stack up, a <strong>Dismiss all</strong> link appears</li>
</ul>

<h2>Notification Center</h2>
<p>Click the <strong>Inbox icon</strong> in the sidebar to open the Notification Center. It has two tabs:</p>
<table>
  <tr><th>Tab</th><th>What it shows</th></tr>
  <tr><td><strong>Alert History</strong></td><td>Every notification since opening the portal (up to 200, stored in your browser). You can acknowledge directly from here.</td></tr>
  <tr><td><strong>Active Problems</strong></td><td>All currently open Zabbix problems. You can acknowledge from here too.</td></tr>
</table>
<p><strong>Mark all as read</strong> — clears the unread badge without deleting history.<br>
<strong>Clear history</strong> — hides current history entries. New notifications will still appear.</p>

<h2>Alert Sounds</h2>
<p>Controls are in the sidebar next to the active problems count:</p>
<ul>
  <li><strong>Volume icon</strong> — mute or unmute all alert sounds</li>
  <li><strong>Music note icon</strong> — choose a preset: Beep, Chime, Ping, or Alarm. Higher severity = more beeps / higher frequency.</li>
  <li>You can also <strong>upload your own audio file</strong> from the sound menu.</li>
</ul>
<div class="tip">Alert sounds are synthesised in the browser — no audio files are downloaded. This works fully offline.</div>

<h1 id="troubleshooting">7. Troubleshooting</h1>

<table>
  <tr><th>Symptom</th><th>Why it happens</th><th>How to fix it</th></tr>
  <tr>
    <td><strong>"Backend API" dot is red</strong></td>
    <td>The frontend cannot reach the Overwatch backend service.</td>
    <td>Check the backend pod is running: <code>kubectl get pods -n &lt;namespace&gt;</code>. Read its logs if crashing: <code>kubectl logs &lt;pod-name&gt;</code>. Locally: make sure the backend server is started.</td>
  </tr>
  <tr>
    <td><strong>"Zabbix" dot is red</strong></td>
    <td>The backend is running but cannot connect to Zabbix.</td>
    <td>Verify <code>ZABBIX_URL</code>, <code>ZABBIX_USER</code>, and <code>ZABBIX_PASS</code> are correct. Try logging in to the Zabbix web UI directly with those credentials. Check backend logs for <code>ZabbixAPI</code> errors.</td>
  </tr>
  <tr>
    <td><strong>"Invalid credentials" on login</strong></td>
    <td>Wrong username or password.</td>
    <td>Check caps lock. Ask an admin to reset your password. Default first-time credentials: username <code>Admin</code>, password <code>admin</code>.</td>
  </tr>
  <tr>
    <td><strong>Page shows no data / tables are empty</strong></td>
    <td>Backend or Zabbix is unreachable, or the host has no configured items.</td>
    <td>Check both status dots are green. Try a full page reload. Confirm the host has items configured in Zabbix.</td>
  </tr>
  <tr>
    <td><strong>"Host agent unreachable" banner</strong></td>
    <td>Zabbix cannot collect data from the host's agent.</td>
    <td>Confirm the host is powered on. Check the Zabbix agent service: <code>systemctl status zabbix-agent</code>. Check firewall rules — Zabbix needs port <strong>10050</strong> (passive) or agent needs port <strong>10051</strong> (active).</td>
  </tr>
  <tr>
    <td><strong>Acknowledged problem — badge did not update</strong></td>
    <td>The badge refreshes every 10 seconds.</td>
    <td>Wait up to 10 seconds. If it still does not update, refresh the page.</td>
  </tr>
  <tr>
    <td><strong>Export downloads empty or corrupted file</strong></td>
    <td>Backend unreachable, or no hosts exist to export.</td>
    <td>Confirm both status dots are green. Try the other format (XLSX vs CSV). If the host list is empty, the export will only contain headers.</td>
  </tr>
  <tr>
    <td><strong>Custom alert rule is not firing</strong></td>
    <td>The check runs every 60 seconds, or the item is not collecting data.</td>
    <td>Wait up to 1 minute after the threshold is crossed. Check Latest Data for that item — it should not show a <strong>No data</strong> chip. Verify the operator and threshold are correct.</td>
  </tr>
  <tr>
    <td><strong>SLA tab is always empty</strong></td>
    <td>No SLAs defined in Zabbix, or Zabbix version is older than 6.0.</td>
    <td>Confirm SLAs exist in Zabbix directly. Verify the Zabbix service account has read access to services.</td>
  </tr>
  <tr>
    <td><strong>Cannot assign a role / option is greyed out</strong></td>
    <td>You are trying to assign a role higher than your own.</td>
    <td>Ask a Root user to make the change.</td>
  </tr>
  <tr>
    <td><strong>Graphs page shows "No graphs available"</strong></td>
    <td>Host has no graph definitions, or the host is unreachable.</td>
    <td>Confirm graph definitions exist in Zabbix (Monitoring → Hosts → Graphs). Confirm the host has templates applied.</td>
  </tr>
</table>

<h2>Still stuck?</h2>
<p>Check these in order:</p>
<ol>
  <li><strong>Backend logs</strong> — <code>kubectl logs &lt;backend-pod&gt; -n &lt;namespace&gt;</code> (or <code>docker logs backend</code> locally).</li>
  <li><strong>Browser console</strong> — open Developer Tools (F12) → Console tab. API errors appear there.</li>
  <li><strong>Network tab</strong> — Developer Tools → Network. Look for red (failed) requests to <code>/api/*</code>. The response body usually contains the error message.</li>
  <li><strong>Zabbix directly</strong> — log in to the Zabbix web UI to confirm whether the issue is in Zabbix or in Overwatch.</li>
</ol>
