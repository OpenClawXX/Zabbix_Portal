OpenShift Secret & ConfigMap — what to add after the review changes
====================================================================

WHAT YOU CURRENTLY HAVE
------------------------
Secret:      ZABBIX_USER, ZABBIX_PASS, SECRET_KEY
ConfigMap:   BACKEND_URL, DATABASE_URL, ZABBIX_URL


WHAT TO ADD
-----------

ConfigMap — add this one new variable:

  ALLOWED_ORIGINS = https://your-frontend-route.apps.cluster.example.com
  (the Route URL of your frontend — no port, no trailing slash)
  This was added today for CORS. Without it the backend defaults to
  allowing all origins (*), which works but is less secure.


NOTHING ELSE IS NEW — all other changes today were code-only and do
not require new environment variables.


RECOMMENDATION (optional, not urgent)
--------------------------------------
DATABASE_URL contains a password (postgresql://user:password@host/db)
so it ideally belongs in the Secret, not the ConfigMap. Your app works
either way — move it when convenient.


FULL REFERENCE — every env var the backend reads
-------------------------------------------------

Secret (sensitive):
  ZABBIX_USER       Zabbix admin username
  ZABBIX_PASS       Zabbix admin password
  SECRET_KEY        JWT signing key — long random string, never share
  DATABASE_URL      PostgreSQL connection string (contains password)

ConfigMap (non-sensitive):
  ZABBIX_URL        URL of your Zabbix server
  BACKEND_URL       Backend URL — consumed by frontend proxy
  ALLOWED_ORIGINS   Frontend Route URL for CORS (added today)

Optional (have safe defaults if not set):
  ADMIN_USERNAME    Root account username on first boot (default: Admin)
  ADMIN_PASSWORD    Root account password on first boot (default: admin — change immediately)
  ZABBIX_SSL_VERIFY Set to false if Zabbix uses a self-signed cert (default: true)
  LOG_LEVEL         Backend log verbosity: DEBUG | INFO | WARNING | ERROR (default: INFO)
